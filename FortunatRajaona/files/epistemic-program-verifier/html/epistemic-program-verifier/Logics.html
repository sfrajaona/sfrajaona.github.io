Programming Language

data Prog Source #

The following is the syntax of the program.

We explain the difference between Assert and Assume (in the context programs without knowledge).

  • Assumption. \[sp(\phi, Assume\ \beta) = \phi \wedge \beta \quad\text{ and }\quad wp(\phi, Assume\ \beta) = \phi \rightarrow \beta\] If an assumption is false, the program blocks (magic). In that case, from an initial set \(S\) of states, the resulting set of state is \(\varnothing\). Intuitively the wp says that anything is true in the empty set of states.
  • Assertion. \[sp(\phi, Assume\ \beta) = \phi \rightarrow \beta \quad\text{ and }\quad wp(\phi, Assume\ \beta) = \phi \wedge \beta\] If an assumption is false, the program fails (error). In that case, from an initial state \(s\), the resulting set of states is error (havoc). Intuitively the sp says that if \(\phi\) is false, any state (including error states) could result from the initial state. Because any \(\beta\) satisfies the sp.

Test in dynamic logic correspond to TestAssume. TestAssume below is a classical test (on FOLFormula), and its sp is defined as a conjunction as for assumption.

Assert and Assume below are defined for ModalFormula, and their wp are not conjunction and implication, but makes use of public announcement formula.

wp :: ModalFormula -> Prog -> ModalFormula Source #

The function wp returns the weakest precondition of a modal formula w.r.t a program. The wp of assumption, assertion, assignment and conditional make use of a public announcement formula. The running program is publicly known.
